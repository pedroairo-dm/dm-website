Today i was importing a crm 2011 solution, when i saw the memory used be the w3wp process, yes it was about 3.3 gbs.

I would say thanks to the 64 bits server technology.

My question is, what could happen if instead of 64 bits server, we have 32 bits server?


Garbage collector  working or crm import typical error, remember that 32bits server only support up to 2 gbs of memory by process.

And i know, the crm 2011 is only supported on 64 bits servers type, but previous versions were/are supported in 32 bits servers type.
